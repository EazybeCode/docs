## Prerequisites

<Check>WABA connected to Eazybe</Check>
<Check>Admin or Manager role</Check>
<Check>Clear use case defined</Check>

## Step-by-Step Guide

<Steps>
  <Step title="Navigate to Agents">
    Go to **Workspace** → **Automation** → **Agents**
  </Step>
  <Step title="Click Create New">
    Click **+ Create Agent** button
  </Step>
  <Step title="Name Your Agent">
    Give it a descriptive name (e.g., "FAQ Bot", "Lead Qualifier")
  </Step>
  <Step title="Set Trigger">
    Choose when the bot activates:
    - All new conversations
    - Outside business hours only
    - Specific keywords
  </Step>
  <Step title="Build Conversation Flow">
    Use the visual builder to create the conversation
  </Step>
  <Step title="Add Human Handoff">
    Always include a way to reach a human
  </Step>
  <Step title="Test">
    Use the preview feature to test all paths
  </Step>
  <Step title="Activate">
    Toggle the agent to **Active**
  </Step>
</Steps>

## Example: Simple FAQ Bot

### Flow Structure

```
[New Conversation]
    ↓
[Welcome Message]
"Hi! I'm the Eazybe assistant. What can I help with?"
    ↓
[Button Options]
├── "Pricing" → Send pricing info
├── "Demo" → Book demo link
├── "Support" → Create ticket
└── "Talk to human" → Handoff
```

### Building This Flow

1. **Start Node**: New conversation trigger
2. **Send Message**: Welcome greeting
3. **Button Node**: Present options
4. **Branch**: Different path for each button
5. **End Node**: Human handoff option

## Tips for Success

<Tip>
  Keep initial menu options to 3-4 choices. Too many options overwhelm users.
</Tip>

### Response Best Practices

| Good | Bad |
|------|-----|
| Short, clear messages | Long paragraphs |
| Actionable options | Vague responses |
| Friendly tone | Robotic language |
| Quick answers | Multiple questions at once |

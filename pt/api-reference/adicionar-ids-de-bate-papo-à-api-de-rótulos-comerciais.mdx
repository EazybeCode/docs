#### Visão Geral:
Esta API permite que você adicione conversas específicas a etiquetas de negócios. Ela verifica se a sessão na nuvem está ativa, recupera as etiquetas de negócios disponíveis e associa as conversas especificadas a uma etiqueta determinada.


#### Endpoint:
**POST http://api.eazybe.com/v2/cloud/business-label**


#### Autenticação:

- **Tipo:** Bearer Token


- A API exige um cabeçalho de Autorização com um token de autenticação válido da Organização para autenticar a solicitação.


### 


#### Como obter o Token de Autenticação da Organização?
Você pode obter o Token de Autenticação da Organização a partir do [Workspace do EazyBe](https://workspace.eazybe.com/) seguindo estes passos:


1. O administrador da organização deve fazer login no Workspace do EazyBe usando suas credenciais.

2. Na página da Organização, o Token de Autenticação da Organização será exibido, com apenas os últimos 4 dígitos visíveis (o restante será ocultado).

3. O administrador pode copiar o Token de Autenticação da Organização e colá-lo na solicitação da API.


#### Método da Solicitação:
**POST**


#### Cabeçalhos:
**Chave**

**Valor**

**Descrição**

**Content-Type**

application/json; charset=UTF-8

Especifica que o formato do corpo da solicitação é JSON.

**Authorization**

Bearer {token}

**Obrigatório**. Usado para autenticar a solicitação. O Token Bearer será o Token de Autenticação da Organização específica.


#### Parâmetros do Corpo da Solicitação:
**Parâmetro**

**Tipo**

**Obrigatório**

**Descrição**

**labelName**

string

Sim

Nome da etiqueta de negócios que você deseja criar.

**contacts**

Array de strings

Sim

Lista de contatos a serem associados à etiqueta.


### Exemplo de Solicitação:
{ 
"labelName": "Clientes Prioritários", 
"contacts": [ 
"911234567891", 
"911234567892", 
"911234567893" 
] 
}
#### Respostas:
200 OK:Indica que a etiqueta foi criada com sucesso e as conversas foram associadas a ela.


**Exemplo de Resposta de Sucesso:**

{ 
  "status": true, 
  "data": 
  { 
    "message": "Contatos adicionados com sucesso." 
  } 
}
#### Práticas de Higiene:

- Certifique-se de que o cabeçalho de Autorização contenha um token de Autenticação válido da Organização.


- Verifique se os contatos são válidos.

Bu kılavuz, yöneticilere Zoho CRM'de profil düzeyinde API erişimini etkinleştirme talimatları sağlar. API erişimini etkinleştirmek, Eazybe'nin Zoho CRM verilerinizle iletişim kurabilmesi için gereklidir.

## Bu Kılavuzu Kimler Kullanmalı

<CardGroup cols={3}>
  <Card title="CRM Yöneticileri" icon="user-shield">
    Kullanıcı izinlerini yöneten Zoho CRM yöneticileri
  </Card>
  <Card title="BT/Entegrasyon Ekipleri" icon="network-wired">
    Sistem entegrasyonlarını kuran ekipler
  </Card>
  <Card title="Geliştiriciler" icon="code">
    API tabanlı entegrasyonları yapılandıran geliştiriciler
  </Card>
</CardGroup>

<Note>
  Zoho CRM'de yalnızca yönetici erişimi olan kullanıcılar profil düzeyindeki güvenlik ayarlarını değiştirebilir.
</Note>

## Ön Koşullar

Devam etmeden önce, aşağıdakilere sahip olduğunuzdan emin olun:

- Zoho CRM'e **yönetici erişimi**
- API çağrıları yapan kullanıcıya hangi **profilin** atandığını bilmek (Standart, Satış veya özel profiller)

## API Erişimini Etkinleştirme Adımları

<Steps>
  <Step title="Zoho CRM'e Giriş Yapın">
    Yönetici kimlik bilgilerini kullanarak Zoho CRM hesabınıza giriş yapın.

    ![Zoho CRM'e Giriş Yapın](/images/1968881807.png)
  </Step>

  <Step title="Kuruluma Erişin">
    Ekranın sağ üst köşesindeki **dişli simgesine** (Setup) tıklayın.

    ![Kuruluma Erişin](/images/1968884377.png)
  </Step>

  <Step title="Güvenlik Kontrolüne Gidin">
    **Security Control**'e gidin ve mevcut seçeneklerden **Profiles**'i seçin.

    ![Güvenlik Kontrolüne Gidin](/images/1968890074.png)
  </Step>

  <Step title="Kullanıcı Profilini Seçin">
    **Profiles** sekmesinde, API kullanıcısına atanan profili bulun ve tıklayın.

    ![Kullanıcı Profilini Seçin](/images/1968891619.png)
  </Step>

  <Step title="Geliştirici İzinlerini Bulun">
    Profil ayarları sayfasının en altına kaydırın ve **Developer Permissions** bölümünü bulun.
  </Step>

  <Step title="API Erişimini Etkinleştirin">
    **Zoho CRM API Access** anahtarını **ON** konumuna getirin. Anahtar etkinleştirildiğinde yeşil renkte görüntülenir.

    ![API Erişimini Etkinleştirin](/images/1968892209.png)
  </Step>

  <Step title="Değişiklikleri Kaydedin">
    Güncellenen izinleri uygulamak için **Save**'e tıklayın.
  </Step>
</Steps>

## Doğrulama

API erişimini etkinleştirdikten sonra, yapılandırmayı doğrulayın:

<AccordionGroup>
  <Accordion title="Profil Durumunu Kontrol Edin">
    Profil, **API Access** durumunu **ON** olarak göstermelidir.
  </Accordion>

  <Accordion title="Kimlik Doğrulamayı Test Edin">
    Kullanıcı kimlik doğrulaması API kimlik bilgileri veya OAuth tokenları ile başarılı olmalıdır.
  </Accordion>

  <Accordion title="API Çağrılarını Doğrulayın">
    API çağrıları artık izin hataları döndürmemelidir.
  </Accordion>
</AccordionGroup>

## Sorun Giderme

<Warning>
  API çağrıları erişimi etkinleştirdikten sonra hala başarısız oluyorsa, aşağıdaki yaygın sorunları kontrol edin:
</Warning>

| Sorun | Çözüm |
|-------|-------|
| Yanlış profil güncellendi | API kullanıcısına atanan doğru profili düzenleyip düzenlemediğinizi doğrulayın |
| Oturum yenilenmedi | Kullanıcıdan çıkış yapmasını ve tekrar giriş yapmasını isteyin |
| IP kısıtlamaları | IP tabanlı erişim kısıtlamalarının API çağrılarını engelleyip engellemediğini kontrol edin |
| OAuth kapsamı | OAuth tokeninin gerekli kapsamlara sahip olduğunu doğrulayın |

## Güvenlik En İyi Uygulamaları

<Tip>
  API erişimini yönetirken bu güvenlik önerilerini izleyin:
</Tip>

- **Seçici şekilde etkinleştirin** - API erişimini yalnızca bunu gerektiren profiller için etkinleştirin
- **Geniş erişimden kaçının** - API erişimini tüm profiller için gereksiz yere etkinleştirmeyin
- **Düzenli denetimler** - Hangi profillerin API erişiminin etkinleştirildiğini periyodik olarak gözden geçirin
- **Kullanılmadığında iptal edin** - Entegrasyonlar artık kullanılmadığında API erişimini devre dışı bırakın

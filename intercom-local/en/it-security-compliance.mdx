### Overview
**Eazybe** is a Chrome extension that enhances WhatsApp Web with productivity and CRM integration features - ideal for sales, support, and customer engagement teams. It overlays functionalities such as:

- CRM mini-views inside WhatsApp Web
- Shared team inboxes
- Contact labels, follow-ups, tasks
- Data sync to users’ Google Drive or CRMs

### Data Handling & Storage
Feature

Details

**Message Storage**

Eazybe **does not store messages**. All chats remain within WhatsApp.

**CRM Integration Data**

Displayed via APIs; **no CRM credentials stored**. OAuth where possible.

**Backups**

Stored in the user’s **Google Drive** (user/workspace-owned).

**Attachment Handling**

Media (e.g., PDFs, images) stored in Google Drive if backup is enabled.

**No Server-Side Storage**

All sensitive data is client-side or within user’s cloud (Drive/CRM).

---

### Encryption & Infrastructure

- **Data in Transit**: Encrypted with TLS/HTTPS
- **Data at Rest**: Backed-up data is encrypted using Google Drive’s built-in encryption
- **Access Controls**: Strict role-based access internally
- **Endpoint Security**: Only authorized devices can access infrastructure
- **Production Firewalls**: Deny-by-default rules active

---

### Chat Backup Defaults (Team Inbox)
When Team Inbox is enabled, **all WhatsApp chats are backed up by default** to your connected Google Drive.

**But you’re in control:**

- Use the **"Don’t Sync" label** to exclude specific contacts.
- Admins can manage sync rules org-wide - including what gets backed up and what doesn’t.

[Learn how to stop sync for specific contacts →](https://help.eazybe.com/en/articles/9863119-can-i-stop-sync-for-some-particular-contacts)

### Cloud WhatsApp Connectivity (Optional)
Eazybe offers a cloud-based session feature that enables scheduled or automated WhatsApp messaging - even when the user’s browser is closed.

**Notes:**

- No WhatsApp data is stored on Eazybe servers
- Cloud sessions are authenticated via WhatsApp’s official **QR code-based login**
- Sessions can be **disconnected anytime** from the extension

Setup Guide → [Connecting Cloud WhatsApp](https://help.eazybe.com/en/articles/9921777-how-to-connect-eazybe-cloud-whatsapp)

Disconnect → [How to revoke Cloud access](https://help.eazybe.com/en/articles/9925430-how-to-disconnect-eazybe-cloud-whatsapp)

### IT Setup Requirements
For a smooth installation and evaluation of Eazybe, your IT team should ensure the following environment and access settings are in place:

Requirement

Details

**Google Chrome Browser**

Eazybe is a Chrome Extension and works only on Chrome

**WhatsApp Web Access**

The user must be logged into WhatsApp Web in the same Chrome profile

**Chrome Extension Permissions**

Allow access to WhatsApp Web tabs for UI overlay and functionality

**CRM Access (Optional)**

API key or OAuth permissions to connect and sync with HubSpot, Zoho, Salesforce, etc.

**Google Drive Access (Optional)**

For backup and Team Inbox – Drive access must be granted by the user or Workspace admin

**Firewall/Network Access**

Allow HTTPS traffic to `web.eazybe.com`, `api.eazybe.com`, and CRM endpoints

**Trial Account (Optional)**

Use a test WhatsApp number or CRM sandbox account for safe evaluation

Eazybe does not require software installation on user machines - just a lightweight Chrome extension install with specific tab-level permissions.

### How to Trial Safely
If your company is unsure, we recommend:

- Using a **temporary WhatsApp number** for testing
- Limiting initial access to test accounts in your CRM.
- Reviewing extension activity via **Chrome extension controls.**

---

### Cancellation/Offboarding
All trial access can be revoked on request. Users can:

- Uninstall the extension at any time
- Disconnect CRM and/or Google Sheets integrations from their respective account dashboards
- Request deletion of synced data from our team (if any)

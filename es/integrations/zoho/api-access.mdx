---
title: "Habilitar Acceso a la API de Zoho CRM"
description: "Aprenda cómo habilitar el acceso a la API de Zoho CRM para perfiles de usuario para permitir la integración con Eazybe y una comunicación de datos fluida."
---

Esta guía proporciona a los administradores instrucciones para habilitar el acceso a la API a nivel de perfil en Zoho CRM. Habilitar el acceso a la API es esencial para que Eazybe se comunique con los datos de su Zoho CRM.

## Quién Debe Usar Esta Guía

<CardGroup cols={3}>
  <Card title="Administradores de CRM" icon="user-shield">
    Administradores de Zoho CRM que gestionan permisos de usuario
  </Card>
  <Card title="Equipos de TI/Integración" icon="network-wired">
    Equipos que configuran integraciones de sistemas
  </Card>
  <Card title="Desarrolladores" icon="code">
    Desarrolladores que configuran integraciones basadas en API
  </Card>
</CardGroup>

<Note>
  Solo los usuarios con acceso administrativo pueden modificar la configuración de seguridad a nivel de perfil en Zoho CRM.
</Note>

## Requisitos Previos

Antes de continuar, asegúrese de tener:

- **Acceso administrativo** a Zoho CRM
- Conocimiento de qué **perfil** está asignado al usuario que realiza llamadas a la API (Estándar, Ventas o perfiles personalizados)

## Pasos para Habilitar el Acceso a la API

<Steps>
  <Step title="Iniciar Sesión en Zoho CRM">
    Inicie sesión en su cuenta de Zoho CRM con credenciales de administrador.

    ![Iniciar Sesión en Zoho CRM](/images/1968881807.png)
  </Step>

  <Step title="Acceder a Configuración">
    Haga clic en el **icono de engranaje** (Configuración) en la esquina superior derecha de la pantalla.

    ![Acceder a Configuración](/images/1968884377.png)
  </Step>

  <Step title="Navegar a Control de Seguridad">
    Vaya a **Security Control** y seleccione **Profiles** de las opciones disponibles.

    ![Navegar a Control de Seguridad](/images/1968890074.png)
  </Step>

  <Step title="Seleccionar el Perfil de Usuario">
    En la pestaña **Profiles**, localice y haga clic en el perfil asignado al usuario de la API.

    ![Seleccionar Perfil de Usuario](/images/1968891619.png)
  </Step>

  <Step title="Encontrar Permisos de Desarrollador">
    Desplácese hasta la parte inferior de la página de configuración del perfil y encuentre la sección **Developer Permissions**.
  </Step>

  <Step title="Habilitar Acceso a la API">
    Active el interruptor **Zoho CRM API Access** en **ON**. El interruptor se muestra en verde cuando está habilitado.

    ![Habilitar Acceso a la API](/images/1968892209.png)
  </Step>

  <Step title="Guardar Cambios">
    Haga clic en **Save** para aplicar los permisos actualizados.
  </Step>
</Steps>

## Verificación

Después de habilitar el acceso a la API, verifique la configuración:

<AccordionGroup>
  <Accordion title="Verificar Estado del Perfil">
    El perfil debe mostrar el estado de **API Access** como **ON**.
  </Accordion>

  <Accordion title="Probar Autenticación">
    La autenticación del usuario debe tener éxito con credenciales de API o tokens OAuth.
  </Accordion>

  <Accordion title="Verificar Llamadas a la API">
    Las llamadas a la API ya no deben devolver errores de permisos.
  </Accordion>
</AccordionGroup>

## Solución de Problemas

<Warning>
  Si las llamadas a la API aún fallan después de habilitar el acceso, verifique los siguientes problemas comunes:
</Warning>

| Problema | Solución |
|----------|----------|
| Perfil incorrecto actualizado | Verifique que editó el perfil correcto asignado al usuario de la API |
| Sesión no actualizada | Pida al usuario que cierre sesión y vuelva a iniciarla |
| Restricciones de IP | Verifique si las restricciones de acceso basadas en IP están bloqueando las llamadas a la API |
| Alcance de OAuth | Verifique que el token OAuth tenga los alcances requeridos |

## Mejores Prácticas de Seguridad

<Tip>
  Siga estas recomendaciones de seguridad al gestionar el acceso a la API:
</Tip>

- **Habilitar selectivamente** - Solo habilite el acceso a la API para perfiles que lo requieran
- **Evitar acceso amplio** - No habilite el acceso a la API para todos los perfiles innecesariamente
- **Auditorías regulares** - Revise periódicamente qué perfiles tienen habilitado el acceso a la API
- **Revocar cuando no se use** - Deshabilite el acceso a la API cuando las integraciones ya no estén en uso

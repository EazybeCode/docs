title: "Habilitar Acesso à API do Zoho CRM"
description: "Aprenda como habilitar o acesso à API do Zoho CRM para perfis de usuário para permitir a integração com o Eazybe e uma comunicação de dados fluida."

Este guia fornece aos administradores instruções para habilitar o acesso à API no nível de perfil no Zoho CRM. Habilitar o acesso à API é essencial para que o Eazybe se comunique com os dados do seu Zoho CRM.

</Note>

## Pré-requisitos

Antes de continuar, certifique-se de ter:

- **Acesso administrativo** ao Zoho CRM
- Conhecimento de qual **perfil** está atribuído ao usuário que faz chamadas à API (Padrão, Vendas ou perfis personalizados)

## Passos para Habilitar o Acesso à API

<Steps>
  <Step title="Fazer Login no Zoho CRM">
    Entre na sua conta do Zoho CRM com credenciais de administrador.

    ![Fazer Login no Zoho CRM](/images/1968881807.png)
  </Step>

  <Step title="Acessar Configuração">
    Clique no **ícone de engrenagem** (Configuração) no canto superior direito da tela.

    ![Acessar Configuração](/images/1968884377.png)
  </Step>

  <Step title="Navegar para Controle de Segurança">
    Vá para **Security Control** e selecione **Profiles** das opções disponíveis.

    ![Navegar para Controle de Segurança](/images/1968890074.png)
  </Step>

  <Step title="Selecionar o Perfil do Usuário">
    Na aba **Profiles**, localize e clique no perfil atribuído ao usuário da API.

    ![Selecionar Perfil do Usuário](/images/1968891619.png)
  </Step>

  <Step title="Encontrar Permissões de Desenvolvedor">
    Role até a parte inferior da página de configurações do perfil e encontre a seção **Developer Permissions**.
  </Step>

  <Step title="Habilitar Acesso à API">
    Ative o interruptor **Zoho CRM API Access** para **ON**. O interruptor fica verde quando habilitado.

    ![Habilitar Acesso à API](/images/1968892209.png)
  </Step>

  <Step title="Salvar Alterações">
    Clique em **Save** para aplicar as permissões atualizadas.
  </Step>
</Steps>

## Verificação

Após habilitar o acesso à API, verifique a configuração:

<AccordionGroup>
  <Accordion title="Verificar Status do Perfil">
    O perfil deve exibir o status de **API Access** como **ON**.
  </Accordion>

  <Accordion title="Testar Autenticação">
    A autenticação do usuário deve ter sucesso com credenciais de API ou tokens OAuth.
  </Accordion>

  <Accordion title="Verificar Chamadas à API">
    As chamadas à API não devem mais retornar erros de permissão.
  </Accordion>
</AccordionGroup>

## Solução de Problemas

<Warning>
  Se as chamadas à API ainda falharem após habilitar o acesso, verifique os seguintes problemas comuns:
</Warning>

| Problema | Solução |
|----------|---------|
| Perfil incorreto atualizado | Verifique se você editou o perfil correto atribuído ao usuário da API |
| Sessão não atualizada | Peça ao usuário para fazer logout e login novamente |
| Restrições de IP | Verifique se restrições de acesso baseadas em IP estão bloqueando as chamadas à API |
| Escopo do OAuth | Verifique se o token OAuth tem os escopos necessários |

## Melhores Práticas de Segurança

<Tip>
  Siga estas recomendações de segurança ao gerenciar o acesso à API:
</Tip>

- **Habilitar seletivamente** - Apenas habilite o acesso à API para perfis que necessitam
- **Evitar acesso amplo** - Não habilite o acesso à API para todos os perfis desnecessariamente
- **Auditorias regulares** - Revise periodicamente quais perfis têm o acesso à API habilitado
- **Revogar quando não usado** - Desabilite o acesso à API quando as integrações não estiverem mais em uso

## What This Feature Does

This feature enables administrators to control which HubSpot properties specific teams can edit across **Contacts**, **Deals**, and **Tickets** modules. By configuring edit permissions, you can ensure team members only modify the fields relevant to their roles.

---

## Who Can Use This Feature

<Note>
Only **Admins** can configure property edit permissions for teams. Regular users cannot access these settings.
</Note>

---

## How to Set Edit Properties for Teams

<Steps>
  <Step title="Select a Team">
    Navigate to the **Settings** panel and choose the team you want to configure from the dropdown menu.

    ![Select Team](/images/1814873973.png)
  </Step>

  <Step title="Select a Module">
    Choose which module's properties you want to configure:
    - **Contacts**
    - **Deals**
    - **Tickets**

    ![Select Module](/images/1814874527.png)
  </Step>

  <Step title="Select Edit Properties">
    Check the boxes next to the properties you want the selected team to be able to edit. Unchecked properties will be read-only for that team.

    ![Select Properties](/images/1814875042.png)
  </Step>

  <Step title="Save Settings">
    Click the **Save** button to store your configuration.

    ![Save Settings](/images/1814876131.png)
  </Step>

  <Step title="Apply the Changes">
    Navigate to **Extensions** and click **Apply** to activate the new permissions.

    ![Apply Changes](/images/1814877048.png)
  </Step>
</Steps>

<Warning>
Settings won't take effect until you click **Apply** in the Extensions section.
</Warning>

## FAQ

<AccordionGroup>
  <Accordion title="Can I assign different permissions for multiple teams?">
    Yes, you can configure unique edit property permissions for each team individually. Simply select a different team and repeat the configuration process.
  </Accordion>

  <Accordion title="Do users need to re-login after changes?">
    Users do not always need to re-login after changes. However, refreshing the CRM page is recommended to ensure the updated permissions are applied correctly.
  </Accordion>

  <Accordion title="Who can make these changes?">
    Only users with **Admin** privileges can configure edit property permissions. If you need access, contact your workspace administrator.
  </Accordion>
</AccordionGroup>

## Visao Geral

Este guia fornece aos administradores instrucoes para habilitar o acesso a API no nivel de perfil no Zoho CRM. Habilitar o acesso a API e essencial para que o Eazybe se comunique com os dados do seu Zoho CRM.

</Note>

## Pre-requisitos

Antes de continuar, certifique-se de ter:

- **Acesso administrativo** ao Zoho CRM
- Conhecimento de qual **perfil** esta atribuido ao usuario que faz chamadas a API (Padrao, Vendas ou perfis personalizados)

## Verificacao

Apos habilitar o acesso a API, verifique a configuracao:

<AccordionGroup>
  <Accordion title="Verificar Status do Perfil">
    O perfil deve exibir o status de **Acesso a API** como **ON**.
  </Accordion>

  <Accordion title="Testar Autenticacao">
    A autenticacao do usuario deve ter sucesso com credenciais de API ou tokens OAuth.
  </Accordion>

  <Accordion title="Verificar Chamadas a API">
    As chamadas a API nao devem mais retornar erros de permissao.
  </Accordion>
</AccordionGroup>

## Solucao de Problemas

<Warning>
  Se as chamadas a API ainda falharem apos habilitar o acesso, verifique os seguintes problemas comuns:
</Warning>

| Problema | Solucao |
|----------|---------|
| Perfil incorreto atualizado | Verifique se voce editou o perfil correto atribuido ao usuario da API |
| Sessao nao atualizada | Peca ao usuario para fazer logout e login novamente |
| Restricoes de IP | Verifique se restricoes de acesso baseadas em IP estao bloqueando as chamadas a API |
| Escopo do OAuth | Verifique se o token OAuth tem os escopos necessarios |

## Melhores Praticas de Seguranca

<Tip>
  Siga estas recomendacoes de seguranca ao gerenciar o acesso a API:
</Tip>

- **Habilitar seletivamente** - Apenas habilite o acesso a API para perfis que necessitam
- **Evitar acesso amplo** - Nao habilite o acesso a API para todos os perfis desnecessariamente
- **Auditorias regulares** - Revise periodicamente quais perfis tem o acesso a API habilitado
- **Revogar quando nao usado** - Desabilite o acesso a API quando as integracoes nao estiverem mais em uso
